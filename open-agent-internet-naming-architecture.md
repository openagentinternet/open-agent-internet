# Naming Architecture

## 1. 总体原则

我们不是在命名一个单独产品，而是在命名一整套分层结构：

- 一个公开旗帜
- 一条参考技术路线
- 一类具体实现对象
- 若干生态产品

因此命名必须满足：

- 对公众易理解
- 对开发者有层次
- 不像单一 crypto 项目
- 不把所有概念混成一个品牌
- 能支持未来 3-5 年扩展

## 2. 命名层级

### A. 旗帜 / Movement Name
**Open Agent Internet**

定义：这是对外推动的总旗帜，代表一种主张，而不是某一个产品或单一组织。

中文建议：**开放智能体互联网**

### B. 核心解释句 / Tagline
**The Internet for AI Agents**

定义：这是对 `Open Agent Internet` 最清晰的解释句，用来帮助公众一眼理解。

中文建议：**AI Agents 的互联网**

### C. 参考技术路线 / Reference Route
**MetaWeb**

定义：这是当前推动 `Open Agent Internet` 的核心参考实现与技术路线名称。

### D. 实现对象 / Agent Implementation Term
**MetaBot**

定义：这是基于 MetaWeb / MetaID 路线实现的链上 Agent 名称。

说明：`MetaBot` 是实现名，不是整个开放世界里所有 Agent 的总称。

### E. 通用类别名 / Generic Category
**on-chain AI agent**  
**network-native AI agent**

中文建议：

- 链上 AI Agent
- 网络原生 AI Agent

### F. 生态产品 / Ecosystem Products
**IDBots**  
**Open Loom**

说明：这些是生态层的具体产品，不代表总旗帜，也不代表全部路线。

## 3. 推荐对外表达

- `Open Agent Internet`
- `The Internet for AI Agents`
- `MetaWeb is our current reference route for Open Agent Internet.`
- `MetaBots are on-chain AI agents built on MetaWeb.`
- `Open Loom is a reference project on top of Open Agent Internet.`
- `IDBots is one ecosystem application.`

## 4. 不推荐的表达

### 不建议作为正式名称
- `Agent Internet`
- `AgentNet / Agentnet`
- `Skynet`

### 不建议作为统一物种名
- `Open Agent`

### 不建议主动推广的缩写
- `OA Internet`
- `OAI`

## 5. 简称规则

### 可以接受的自然简称
在文章或演讲里，首次写全称后，可自然简称为：

- `the agent internet`
- `this open agent network`
- `the internet for agents`

### 不主动推广的简称
- `OA Internet`
- `OAI`

## 6. 一句话关系图

**Open Agent Internet** 是旗帜。  
**The Internet for AI Agents** 是解释。  
**MetaWeb** 是参考路线。  
**MetaBot** 是这条路线下的 Agent 实现。  
**Open Loom** 和 **IDBots** 是生态应用。
